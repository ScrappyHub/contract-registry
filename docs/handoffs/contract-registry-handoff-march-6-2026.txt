1 — WHAT THIS PROJECT IS (TO SPEC)

Contract Registry is a deterministic legal artifact registry and release instrument.

It produces, verifies, and records canonical contract release packets that represent:

a contract definition

its schema

policy overlays

the effective policy set applied to the release

deterministic receipts proving what rules were in effect

The registry does not mutate contract truth.
It produces verifiable artifacts whose correctness can be independently proven.

Core guarantees:

deterministic packet generation

Packet Constitution v1 Option-A compliance

canonical bytes (UTF-8 no BOM + LF)

deterministic hashing

non-mutating verification

append-only receipts

This allows contract artifacts to be:

legally verifiable

cryptographically reproducible

ecosystem-portable

2 — TIER-0 SCOPE

Tier-0 is standalone instrument mode.

It proves that the registry can:

build a contract release packet

verify that packet

compare it to golden vectors

produce deterministic receipts

bind the effective policy/schema set used during the release

No network or SaaS infrastructure is required.

4 — PACKET MODEL

Contract releases are produced as directory packets.

Example output:

golden/tmp_outbox/
   contract_release_example.contract.v1_20260101_000000/
       manifest.json
       packet_id.txt
       sha256sums.txt
       receipts/
           release_receipt.txt

Packet invariants:

manifest.json MUST NOT contain packet_id
packet_id.txt = SHA256(canonical manifest bytes)
sha256sums.txt generated LAST
verification is non-mutating
5 — EFFECTIVE SETS MODEL (OVERLAY LAW)

The registry distinguishes between:

Base law
canonical contract schema
canonical policy bundle
Overlay law
policy overlays
schema overlays
environment overlays

During Tier-0 execution the runner resolves:

effective_sets/
   policy_bundle.json
   schema_bundle.json
   receipt.txt

The effective set receipt hash is then bound into the Tier-0 receipt.

Example Tier-0 receipt lines:

effective_sets_receipt_sha256: <sha256>
effective_sets_receipt_path: effective_sets/receipt.txt

This proves:

what rule set governed the contract release

without mutating the contract packet.
